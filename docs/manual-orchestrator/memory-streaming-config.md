# 上下文与记忆、流式输出、配置与安全

## 9. 上下文与记忆设计

### 9.1 当前问题

当前 `checkpoints/context.txt` 仅存储：

- 时间戳
- role
- content

无法表达：

- 工具调用
- 中间轮次
- 结构化参数
- 错误信息

### 9.2 建议演进方向

建议把会话存储升级为结构化格式，优先推荐 JSON Lines。

单条记录可包含：

- `timestamp`
- `session_id`
- `turn`
- `role`
- `message_type`
- `content`
- `tool_name`
- `tool_args`
- `tool_result`
- `trace_id`

### 9.3 历史恢复策略

建议在恢复历史时按层次拼装：

1. 固定系统提示
2. 历史用户/助手消息
3. 近期工具调用摘要
4. 当前用户消息

必要时增加窗口裁剪，例如仅保留最近 N 轮和摘要消息。

## 10. 流式输出设计

### 10.1 目标

虽然不再使用内置 Agent，但仍可保留较好的终端体验。

### 10.2 建议实现方式

分两种模式：

1. **工具决策轮使用 `Generate`**  
   适合需要稳定获取 `tool_calls` 并解析参数时使用。

2. **最终回答轮使用 `Stream`**  
   当编排器已经确定无需继续调用工具时，可让模型直接流式输出最终文本。

### 10.3 推荐策略

短期建议优先稳定性：

- 所有决策轮统一使用 `Generate`
- 仅最终回答阶段使用 `Stream`

这样能避免边流式接收边做结构化解析带来的复杂度。

## 11. 配置与安全治理

### 11.1 配置项外置

建议将以下信息移出源码：

- API Key
- Base URL
- 模型名
- 最大轮次
- 工具超时时间
- 是否启用某个工具

### 11.2 推荐配置来源

优先顺序建议为：

1. 环境变量
2. 本地配置文件
3. 默认值

### 11.3 关键建议

`agent/agent.go` 中的硬编码密钥必须移除，否则代码仓库存在直接泄漏风险。
